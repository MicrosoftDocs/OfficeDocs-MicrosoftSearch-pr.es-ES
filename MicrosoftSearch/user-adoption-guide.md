# <a name="user-adoption-guide"></a>Guía de adopción del usuario

Esta información está dirigida a los administradores de Búsqueda de Microsoft, coordinadores de administración de cambios y propietarios de empresas.
  
Para implementar y garantizar la adopción satisfactoria de Búsqueda de Microsoft en su organización, recomendamos este proceso de cuatro fases.
  
## <a name="stage-1-stakeholders"></a>Fase 1: partes interesadas

Para garantizar un lanzamiento sin problemas y aumentar la adopción por parte del usuario de Búsqueda de Microsoft, identifique a las personas que pueden ayudar a proporcionar soporte técnico y a dirigir el proceso.
  
La tabla muestra la jerarquía del equipo y en lo que contribuye cada persona. 
  
 **Roles y responsabilidades del equipo de adopción de usuarios**
  
|**Rol**|**Responsabilidades**|**Departamento**|
|:-----|:-----|:-----|
|Patrocinador Ejecutivo   <br/> |Comprender la visión y el valor de alto nivel de Búsqueda de Microsoft para su organización   <br/> Proporcionar soporte de liderazgo al equipo de TI o la empresa   <br/> Asignar recursos  <br/> Comunicarse directamente con los empleados  <br/> |Liderazgo ejecutivo  <br/> |
|Propietario de empresa  <br/> |Identificar los escenarios de producto y usuario   <br/> Determinar las métricas de éxito para los departamentos y la organización   <br/> |Cualquiera  <br/> |
|Equipo de administración de cambios  <br/> |Garantizar que el valor del producto es fácil de entender  <br/> Crear un plan para la implementación   <br/> |TI  <br/> |
|Expertos:  <br/> |Promover el producto entre los compañeros y equipos  <br/> |Cualquiera (personal)  <br/> |
|Administrador de búsqueda   <br/> |Identificar a los miembros principales del equipo de adopción del usuario  <br/> Administrar Búsqueda de Microsoft, incluida la apariencia de la experiencia y la administración continuada   <br/> Impulsar discusiones para mostrar el valor de Búsqueda de Microsoft   <br/> |TI  <br/> |
|Editors  <br/> |Crear contenido en el portal de administración   <br/> |TI/Cualquiera  <br/> |
   
## <a name="stage-2-scenarios"></a>Fase 2: escenarios

Para garantizar la adopción por parte del usuario, es fundamental determinar y definir los escenarios del usuario final. Estos deben incluir ejemplos reales de cómo los usuarios pueden usar Búsqueda de Microsoft para encontrar la información que necesitan, responder preguntas y completar las tareas con mayor rapidez. Para medir el éxito de su implementación y la adopción por parte de los usuarios, también es importante determinar las métricas que usará antes de empezar.
  
 **Comprender lo que los usuarios necesitan y quieren**
  
Cuando se trata de buscar información y obtener respuestas, es fundamental conocer las necesidades y los principales problemas para los usuarios. Puede usar métodos cuantitativos, como datos de Office 365 y el uso de búsqueda de SharePoint, o métodos cualitativos, como encuestas y grupos de enfoque del usuario, para obtener estos datos. Esto le ayudará a determinar cuándo, dónde y cómo Búsqueda de Microsoft puede proporcionar mayor valor.
  
Para varios roles y equipos en la organización, tenga en cuenta lo siguiente:
  
- Los problemas existentes de los usuarios al buscar información.
    
- Aplicaciones, herramientas, sitios y otros orígenes de información que se usan con frecuencia.
    
- Lo que los usuarios desearían ver cuando se trata de buscar información.
    
- Cómo se aplican las ventajas de Búsqueda de Microsoft a su organización. Para obtener más información, vea [¿Por qué Búsqueda de Microsoft?](why-microsoft-search.md).
    
Elija los escenarios de usuario que tendrán el mayor impacto positivo para los usuarios.
  
 **Identificar métricas de éxito**
  
Para ayudarle a cumplir con los objetivos de implementación y mostrar un impacto positivo para su organización, es fundamental determinar las métricas de éxito clave antes de empezar. Mantenga sus objetivos simples y medibles y evalúe el rendimiento de forma regular para que pueda responder rápidamente si es necesario.
  
1. **Administración de cambios**: determine cómo se medirá el conocimiento sobre Búsqueda de Microsoft y el éxito de las comunicaciones. 
    
2. **Uso:** puede usar los datos de su panel de información, incluidos los usuarios activos diarios y mensuales (DAU y MAU), los tipos de contenido que reciben la mayoría de las visitas y las principales palabras clave de búsqueda para medir el uso a lo largo del tiempo. Para obtener más información, consulte [Usar el panel de información](get-insights.md).
    
3. **Satisfacción del usuario**: use los datos que recopiló de las encuestas y los grupos de enfoque del usuario para determinar la línea base. La satisfacción neta (NSAT) y la Net Promoter Score (NPS) también pueden ofrecer mediciones útiles. Independientemente del método que use, manténgalo coherente para que pueda ver la repercusión a lo largo del tiempo. 
    
4. **Otras mediciones:** use datos históricos y/o actuales para las solicitudes de soporte técnico y la cantidad de tiempo invertido en la búsqueda. 
    
## <a name="stage-3-readiness"></a>Fase 3: preparación

Invertir tiempo en la preparación del lanzamiento de Búsqueda de Microsoft supondrá un impacto positivo en la organización. 
  
 **Preparación del producto y el contenido**
  
Los administradores pueden influir directamente en la experiencia de búsqueda para los usuarios finales y determinar el aspecto de Búsqueda de Microsoft para la organización. Esto incluye la elección de los tipos de resultados que quiere que se muestren a los usuarios. Puede controlarlo activando o desactivando los resultados de documentos, sitios, personas, grupos o conversaciones.
  
Para obtener más información, vea [Configurar Búsqueda de Microsoft](set-up-microsoft-search.md).
  
Los editores son expertos en la materia que sirven para administrar Marcadores y preguntas frecuentes en sus áreas de especialización. Los editores deberían crear contenido que esté dirigido a los escenarios de usuario identificados anteriormente.
  
Para obtener más información, vea [Planificar el contenido](plan-your-content.md).
  
 **Iniciar sesión**
  
Los usuarios de Búsqueda de Microsoft tienen que iniciar sesión en Bing con una cuenta profesional o educativa. Para que los usuarios puedan iniciar sesión lo más fácilmente posible:
  
- Asegúrese de que las configuraciones de Azure Active Directory estén correctamente establecidas, haga que algunos usuarios intenten iniciar sesión. Para obtener más información, vea [Prueba de inicio de sesión único](test-single-sign-on.md).
    
- Establezca los valor predeterminados del explorador para dispositivos administrados por la empresa a Bing. Para obtener más información, consulte [Establecer el explorador predeterminado](set-default-browser.md).
    
- Implemente extensiones de búsqueda para usuarios de Microsoft Edge y Chrome a través de la Directiva de grupo. Para más información, vea [Herramientas del portal de administración](admin-portal-tools.md).
    
 **Preparar comunicaciones**
  
Use los escenarios reales para informar a los usuarios sobre Búsqueda de Microsoft y cómo les ayudará a ahorrar tiempo.  El equipo de administración de cambios puede ayudarle a crear una estrategia de implementación y a preparar materiales de marketing y campañas para la preparación del usuario final.
  
Empiece a planear las comunicaciones y los eventos de su organización:
  
- Pancartas, carteles y folletos ubicados en lugares destacados
    
- Demostraciones del producto en persona o en estand
    
- Eventos en línea para equipos locales y remotos
    
- Haga que el patrocinador ejecutivo lleve a cabo una reunión general o envíe un correo electrónico como anuncio
    
- Comparta documentación de autoayuda sobre el uso de Búsqueda de Microsoft
    
Todas las comunicaciones deben instruir a los usuarios sobre cómo pueden iniciar sesión en Bing para buscar resultados de trabajo.
  
 **Involucrar a los equipos asociados**
  
Como parte integral de Microsoft 365, Búsqueda de Microsoft cumple con las directrices de GDPR y Tier C. Según la organización, puede que sea necesario compartir información acerca de Búsqueda de Microsoft con los equipos de seguridad y privacidad. Para obtener más información, vea [Seguridad para Búsqueda de Microsoft](security.md).
  
Asegúrese de que el servicio de asistencia técnica conozca las características y funciones de Búsqueda de Microsoft, así como los requisitos de usuario para iniciar sesión, de modo que puedan ayudar a los usuarios con más eficiencia. Para obtener más información, vea [Usar Búsqueda de Microsoft](use/about-microsoft-search.md) y [Preguntas frecuentes](faqs.md).
  
 **Evaluar con usuarios piloto**
  
Empiece con un pequeño grupo de usuarios para evaluar el impacto de Búsqueda de Microsoft y recopilar comentarios anticipados. Esto le proporciona tiempo para cambiar el contenido, actualizar las comunicaciones y establecer expectativas para la implementación general. Los usuarios avanzados de búsqueda o los primeros usuarios son buenos candidatos para esto.
  
Para ayudarle a encontrar los usuarios piloto:
  
- Asociarse con un patrocinador empresarial de su organización
    
- Identificar un grupo de primeros usuarios
    
- Identificar los grupos que realizan grand cantidad de búsquedas de empresa
    
- Identificar los usuarios de otras ubicaciones geográficas, incluidas las internacionales
    
Pida a los usuarios piloto que proporcionen datos con las opciones de comentarios de Búsqueda de Microsoft o a través de encuestas sencillas.
  
Preguntas para encuestas: 
  
- ¿Entiende cómo iniciar sesión y usar Búsqueda de Microsoft?
    
- ¿Funciona como esperaba?
    
- ¿Qué características le gustan?
    
- ¿Ha podido encontrar resultados de trabajo?
    
- ¿Eran útiles los resultados de trabajo?
    
- ¿Las comunicaciones sobre Búsqueda de Microsoft son claras y fáciles de comprender?
    
Este es un buen momento para comenzar a medir la efectividad de las métricas identificadas en la fase 2.
  
Cuando comience la fase piloto, puede considerar compartir un anticipo de Búsqueda de Microsoft con el resto de la organización.
  
## <a name="stage-4-roll-out"></a>Fase 4: implementación

Cada organización piensa en las implementaciones de productos de forma diferente. A continuación, se exponen algunos aspectos a tener en cuenta:
  
 **Amplia implementación**
  
Cuando esté listo, puede permitir que todos los usuarios de la organización obtengan acceso a Búsqueda de Microsoft o usar un enfoque por fases para implementarlo más despacio. 
  
 **Generar conocimiento**
  
Cree entusiasmo en su organización sobre Búsqueda de Microsoft:
  
- Inscriba a los usuarios piloto para ofrecer asistencia y experiencia a sus compañeros.
    
- Realice un seguimiento de las comunicaciones y eventos que planificó en la fase 3.
    
 **Comunicación constante**
  
Durante las primeras semanas y meses, conserve un ritmo de comunicaciones para alentar el uso de Búsqueda de Microsoft. Esto mantiene informados a los usuarios y atrae a nuevos usuarios. Todas las comunicaciones deben explicar por qué Búsqueda de Microsoft puede ser útil para su trabajo.
  
Envíe comunicaciones constantes que resalten las características y las búsquedas para probar, así como las historias de éxito de usuarios.
  
 **Recopilar comentarios y mejorar**
  
Ayude a los usuarios y haga ver que se interesa escuchando y actuando en función de sus comentarios.
  
Use el panel de información para ver cómo funcionan el producto y el contenido. Aproveche esta información para actualizar marcadores, preguntas y respuestas y otras comunicaciones. Esto también le ayudará a:
  
1. Mejorar la calidad del contenido: revisar títulos, descripciones y URL, optimizar palabras clave, quitar información obsoleta y agregar nueva información
    
2. Determinar las características que se usan con mayor o menor frecuencia
    
3. Llevar a cabo encuestas, grupos de enfoque y sesiones de comentarios para comprender lo que los usuarios piensan sobre Búsqueda de Microsoft, aumentar la adopción e identificar áreas que mejorar.
    
El aprendizaje y los comentarios son un proceso iterativo. Puede que sea necesario realizar cambios en el contenido y en las comunicaciones a medida que los usuarios adopten Búsqueda de Microsoft.

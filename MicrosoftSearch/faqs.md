# <a name="faqs"></a>Preguntas más frecuentes

Estas son las preguntas más frecuentes que las organizaciones y los usuarios tienen sobre cómo administrar y usar Microsoft Search.
  
## <a name="whats-the-value-proposition-for-microsoft-search"></a>¿Cuál es la propuesta de valor para Microsoft Search?

Microsoft Search es una forma segura para buscar contenido en la web y su trabajo. Este tipo de integración con la web y la empresa solo está disponible con Microsoft. Para obtener información relacionada con el cumplimiento, consulte [Seguridad de Microsoft Search](security.md).
  
## <a name="what-microsoft-search-features-are-available-now"></a>¿Qué características de Microsoft Search están disponibles en este momento?

Para obtener una lista completa, vea [Características de Microsoft Search](features.md).
  
## <a name="does-microsoft-search-support-advanced-query-understanding"></a>¿Admite Microsoft Search la descripción de consultas avanzadas?

Sí, Microsoft Search analiza el objetivo de la consulta de frases más grandes. Esta característica usa inteligencia artificial para aprender frases superfluas y comunes que los usuarios agregan a sus consultas y que no afectan a su intención de búsqueda. Por ejemplo, cuando un usuario busca "más información sobre cómo cambiar la contraseña" se extraen las palabras menos importantes de la consulta y se busca con las palabras relevantes como "Cambiar contraseña".
  
Esta característica no reemplazará el conjunto de palabras clave en el portal de administración.
  
## <a name="does-microsoft-search-search-for-files-on-premises-as-well-as-the-cloud"></a>¿Microsoft Search busca archivos locales además de en la nube?

Microsoft Search permite buscar documentos que residan en SharePoint Online y OneDrive para la Empresa, así como en SharePoint híbrido local y SharePoint Online para los tipos de archivo más comunes de Office. Si puede buscar contenido local de SharePoint Online, debería poder encontrar contenido local con Microsoft Search. 
  
## <a name="does-microsoft-search-replace-other-enterprise-search-experiences"></a>¿Microsoft Search reemplaza otras experiencias de búsqueda empresarial?

Microsoft Search es una oferta complementaria que reúne los resultados de varios orígenes. Para más información sobre los orígenes y tipos de archivo admitidos, consulte las preguntas más frecuentes siguientes.
  
## <a name="what-file-types-and-sources-does-microsoft-search-support"></a>¿Qué tipos de archivos y orígenes admite Microsoft Search?

Se admiten los siguientes orígenes de contenido:
  
- SharePoint Online
    
- SharePoint híbrido (local y SPO)
    
- OneDrive para la Empresa
    
Los siguientes tipos de archivo se muestran en la búsqueda de archivos y aparecen en la pestaña de archivos para usuarios y grupos:
  
- Word
    
- Excel
    
- PowerPoint
    
- OneNote
    
- PDF

Para obtener información sobre las medidas de cumplimiento y confianza, vea [Seguridad de Microsoft Search](security.md).
  
## <a name="where-can-i-get-info-about-office-365-compliance-tierscategories"></a>¿Dónde puedo obtener información sobre los niveles o categorías de cumplimiento de Office 365?

Encontrará más detalles en el [Marco de cumplimiento para las regulaciones y estándares del sector](https://download.microsoft.com/download/B/2/7/B27B3EF3-8849-4C18-8BA4-5AD755728620/Compliance%20Framework_customer%20guidance.pdf) (descargar PDF). 
  
## <a name="how-does-microsoft-search-keep-results-secure"></a>¿Cómo protege Microsoft Search los resultados?

Para obtener información sobre cómo Microsoft Search mantiene la seguridad de los resultados, consulte [Seguridad de Microsoft Search](security.md).
  
## <a name="what-are-the-content-sources-for-the-people-card"></a>¿Cuáles son los orígenes de contenido de la tarjeta de contactos?

La tarjeta de contactos deriva información de Azure Active Directory, Exchange Online y SharePoint Online.
  
## <a name="do-microsoft-search-users-earn-microsoft-rewards"></a>¿Los usuarios de Microsoft Search ganan Microsoft Rewards?

Las búsquedas de Microsoft Search no otorgan puntos de Microsoft Rewards y no están asociadas con la cuenta de Microsoft del usuario.
  
## <a name="does-microsoft-search-respect-existing-file-permissions"></a>¿Respeta Microsoft Search los permisos de archivo existentes?

Microsoft Search respeta la restricción de seguridad establecida en el origen. Los usuarios solo pueden ver la información a la que tienen acceso.

## <a name="how-does-microsoft-search-use-azure-active-directory"></a>¿Cómo usa Microsoft Search Azure Active Directory?

Microsoft Search usa Azure Active Directory para autenticar y autorizar el acceso a los datos de su empresa. Microsoft Search cumple con los estándares sobre los procedimientos de control de identidades de todos los productos de Microsoft. Esto significa que los usuarios pueden iniciar sesión automáticamente con el inicio de sesión único.

## <a name="does-microsoft-search-provide-search-results-across-tenants"></a>¿Proporciona Microsoft Search resultados de búsqueda entre distintos espacios empresariales?

No, no se admite el acceso compartido entre espacios empresariales distintos.

## <a name="why-am-i-signed-into-bing-automatically"></a>¿Por qué se inicia mi sesión en Bing automáticamente?

Microsoft Search usa el inicio de sesión único de AAD para iniciar sesión automáticamente con su cuenta profesional o educativa cuando haya iniciado sesión en Windows 10 o una aplicación de Office 365. Puede cerrar sesión en cualquier momento.

Microsoft Search utiliza algoritmos de clasificación inteligente para ordenar las tarjetas de resultados según su relevancia.
